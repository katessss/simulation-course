## Задача

Реализовать генерацию случайного события из конечного дискретного множества двумя методами:

- **Равномерная выборка** — все исходы равновероятны P(Aᵢ) = 1/n
- **Алгоритм Моисеева** — произвольное распределение {p₁, …, pₘ}, Σpₖ = 1

**Часть 1:**  
Приложение «Скажи “да” или “нет”».

**Часть 2:**  
Приложение «Шар предсказаний» (Magic 8-Ball).

## Алгоритм Моисеева

### Идея

Отрезок [0, 1) разбивается на m смежных частей длиной p₁, p₂, …, pₘ. Случайное число ξ ~ U[0, 1) попадает в k-й отрезок с вероятностью pₖ.

```
[0 ──── p₁ ──── p₁+p₂ ──── … ──── 1)
  A₁        A₂               Aₘ

A := ξ  (ξ ~ U[0, 1))
k := 1
loop:
    A := A − pₖ
    if A ≤ 0 → return Aₖ
    k := k + 1
```

### Математическое обоснование

Событие Aₖ наступает тогда и только тогда, когда ξ попало в k-й отрезок:

```
p₁ + … + p_{k−1} < ξ ≤ p₁ + … + pₖ
```

Это эквивалентно условию `A ≤ 0` после k-го вычитания, так как:

```
A = ξ − p₁ − p₂ − … − pₖ ≤ 0
⟺  ξ ≤ p₁ + p₂ + … + pₖ
```

Вероятность попасть именно в этот отрезок равна его длине — pₖ. Алгоритм не хранит кумулятивные суммы явно — они вычисляются на лету через последовательные вычитания.

### Связь с равномерным случаем

При pₖ = 1/m для всех k алгоритм Моисеева полностью эквивалентен `random.choice` — отрезки одинаковы, вероятность каждого равна 1/m. Равномерная выборка — частный случай.

## Реализация
### `moiseyev_generate`

```python
def moiseyev_generate(answers: list[tuple[str, float]]) -> str:
    A = random.random()           # ξ ~ U[0, 1)
    for name, p_k in answers:
        A -= p_k
        if A <= 0:
            return name
    return answers[-1][0]         # страховка от float-погрешности
```

`return answers[-1][0]` — защита от ситуации, когда из-за погрешностей вещественной арифметики (`0.1 + 0.2 + 0.7 ≠ 1.0` точно) A остаётся чуть выше нуля после прохода всех событий.

### Валидация

Перед генерацией проверяется:

- поля «Событие» не пустые
- вероятности — числа ≥ 0
- |Σpₖ − 1| < 0.001

Допуск 0.001 позволяет вводить округлённые значения (0.333 вместо 1/3) без потери корректности.

```python
total = sum(p for _, p in answers)
if abs(total - 1.0) > 0.001:
    messagebox.showerror("Ошибка", f"Сумма = {total:.4f}, должна быть 1.0")
    return
```

## Выводы

- Алгоритм Моисеева корректно воспроизводит заданное дискретное распределение: вероятность выбора события Aₖ точно равна pₖ в силу геометрической интерпретации.
- Равномерная выборка — частный случай алгоритма при pₖ = 1/m; оба метода дают одинаковое распределение.
- Алгоритм требует единственного вызова `random.random()` на одну генерацию и одного прохода по списку — сложность O(m) по числу событий.
- Страховочная строка и допуск на сумму обязательны при работе с вещественными числами — без них программа даёт сбои на корректных с точки зрения пользователя входных данных.
