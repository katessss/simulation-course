# Лабораторная работа #3: Клеточные автоматы. Лесные пожары.

## 1. Цель работы

Реализовать моделирование возникновения и распространения лесных пожаров с использованием двумерного клеточного автомата, реализовав не менее трёх дополнительных правил поведения системы, а также разработать графический интерфейс пользователя для визуализации и управления симуляцией.

---

## 2. Описание модели

### 2.1. Состояния клеток

Каждая клетка двумерной сетки размером **60 × 60** может находиться в одном из пяти состояний:

| Константа      | Значение | Описание         |
|----------------|----------|------------------|
| `STATE_EMPTY`  | 0        | Пустая земля     |
| `STATE_TREE`   | 1        | Дерево           |
| `STATE_BURNING`| 2        | Горящее дерево   |
| `STATE_ASH`    | 3        | Пепел            |
| `STATE_WATER`  | 4        | Вода (река)      |

### 2.2. Базовые правила перехода

Основные правила классического автомата «Лесной пожар»:

1. **Пустая клетка** → с вероятностью `p_growth` вырастает дерево.
2. **Дерево** → с вероятностью `p_lightning` воспламеняется от молнии; также может загореться от горящего соседа с вероятностью, зависящей от возраста дерева.
3. **Горящая клетка** → на следующем шаге превращается в пепел.
4. **Пепел** → с вероятностью `p_ash_blow = 0.05` рассеивается (клетка становится пустой).
5. **Вода** → состояние не меняется.

---

## 3. Дополнительные правила поведения системы

### 3.1. Возраст дерева и горючесть

У каждого дерева отслеживается возраст в тактах (`age`). Это влияет на вероятность воспламенения от соседнего огня:

- **Молодое дерево** (возраст < 20 тактов): вероятность возгорания **снижается на 0.20** - молодые деревья содержат больше влаги.
- **Старое дерево** (возраст > 60 тактов): вероятность возгорания **повышается на 0.20** - старые деревья более сухие и смолистые.

```python
burn_chance = 0.60
if self.age[y][x] < 20:
    burn_chance -= 0.20
elif self.age[y][x] > 60:
    burn_chance += 0.20
```

### 3.2. Влажность воздуха

Параметр `humidity` (от 0.0 до 1.0) задаётся ползунком. Он снижает вероятность распространения огня:

```python
burn_chance -= humidity * 0.80
```

При максимальной влажности (`humidity = 1.0`) вероятность возгорания уменьшается на 0.80, что практически блокирует распространение пожара в обычных условиях.

### 3.3. Дождь

По нажатию кнопки **«Дождь»** активируется событие осадков на случайное время от 1 до 20 секунд. Во время дождя:

- Вероятность распространения огня **снижается на 0.50**.
- Существующие горящие клетки с вероятностью **80%** не воспламеняют соседей.

```python
if self.rain_active:
    burn_chance -= 0.50
```

### 3.4. Защитный эффект воды (реки)

Клетки с водой (`STATE_WATER`) не горят и создают **барьер**, снижающий вероятность перехода огня через реку. Если среди соседей горящей клетки есть вода - шанс возгорания дополнительно снижается:

```python
if STATE_WATER in neighbors:
    burn_chance -= 0.15
```

Реки генерируются процедурно с небольшой извилистостью и пересекают поле горизонтально и вертикально.

### 3.5. Рассеивание пепла

После тушения пожара клетки не остаются пеплом бесконечно. С вероятностью `p_ash_blow = 0.05` за каждый такт пепел рассеивается, освобождая место для нового роста леса. Это обеспечивает цикличность экосистемы.

## 4. Генерация мира

При запуске и сбросе симуляции автоматически генерируется мир:

- **Лес**: каждая клетка заполняется деревом с вероятностью **0.62**; возраст - случайное значение от 0 до 180 тактов.
- **Реки**: 4 реки (2 горизонтальных, 2 вертикальных) генерируются с шириной 1–2 клетки и случайной извилистостью.

## 5. Графический интерфейс

Приложение реализовано на **Python + tkinter**.

### 5.1. Структура интерфейса

- **Холст (Canvas)** 600 × 600 пикселей - основная область симуляции.
- **Боковая панель** - управление симуляцией.

### 5.2. Элементы управления

| Элемент                  | Назначение                                              |
|--------------------------|---------------------------------------------------------|
| Кисти (5 инструментов)   | Рисование на поле: лес, вода, огонь, ластик, курсор    |
| Ползунок «Рост деревьев» | Вероятность появления нового дерева на пустой клетке   |
| Ползунок «Шанс молнии»   | Вероятность самовозгорания дерева                      |
| Ползунок «Влажность»     | Влияет на скорость распространения огня                |
| Кнопка «Старт / Пауза»   | Запуск и остановка симуляции                           |
| Кнопка «Весь лес»        | Заполнить всё поле деревьями                           |
| Кнопка «Очистка»         | Очистить поле                                          |
| Кнопка «Дождь»           | Активировать случайный дождь                           |
| Кнопка «Сброс»           | Сгенерировать новый мир                                |
| Статистика               | Счётчики деревьев и горящих клеток с индикаторами      |
| Таймер                   | Отображение времени и номера текущего такта            |

### 5.3. Цветовая схема

| Состояние        | Цвет                        |
|------------------|-----------------------------|
| Пустая земля     | Светло-серый `#F9FAFB`      |
| Молодое дерево   | Светло-зелёный `#86EFAC`    |
| Взрослое дерево  | Зелёный `#22C55E`           |
| Старое дерево    | Тёмно-зелёный `#166534`     |
| Горение          | Красный / жёлтый (мерцание) |
| Пепел            | Серый `#D1D5DB`             |
| Вода             | Синий `#3B82F6`             |

## 8. Выводы

В ходе работы был реализован симулятор лесного пожара на основе двумерного клеточного автомата. Помимо базовой модели «пустая земля - дерево - огонь - пепел», реализованы следующие **дополнительные правила**:

1. **Зависимость горючести от возраста дерева** - молодые деревья горят хуже, старые лучше.
2. **Влажность воздуха** - параметризованное снижение скорости распространения огня.
3. **Дождь** - временное событие, резко снижающее вероятность возгорания.
4. **Защитный барьер рек** - водные клетки замедляют распространение пожара.
5. **Рассеивание пепла** - обеспечивает цикличность экосистемы.

Симуляция демонстрирует реалистичное поведение: при высокой влажности и наличии рек пожары локализуются, при сухих условиях - охватывают большую часть леса. Интерактивный интерфейс позволяет в реальном времени наблюдать и управлять динамикой системы.
