# Лабораторная работа №4: Базовый датчик случайных чисел

## 1. Цель работы

- Реализовать собственный датчик случайных чисел на основе хэш-функции.
- Вычислить выборочные среднее и дисперсию для реализованного и встроенного генераторов.
- Сравнить полученные результаты с теоретическими значениями.
- Сделать вывод о качестве реализованного генератора.

Размер выборки: **N = 100 000 значений**.

---

## 2. Теоретическая база

Для равномерного распределения на отрезке $[0, 1]$ - обозначается $U(0, 1)$ - теоретические характеристики:

$$\mathbb{E}[X] = \frac{a + b}{2} = \frac{0 + 1}{2} = 0{,}5$$

$$\mathbb{D}[X] = \frac{(b - a)^2}{12} = \frac{1}{12} \approx 0{,}0833$$

Именно эти значения являются эталонными при проверке качества генератора.

---

## 3. Реализация собственного генератора

### 3.1. Общая архитектура

Генератор построен по принципу **CTR-подобной схемы** (Counter Mode): на основе начального значения (`seed`) формируется фиксированный ключ, который затем совместно с монотонно возрастающим счётчиком подаётся на вход хэш-функции. Полученный 32-битный результат нормируется в отрезок $[0, 1]$.

Каждое псевдослучайное число однозначно определяется парой «ключ + номер вызова», что обеспечивает воспроизводимость последовательности при одинаковом `seed`.

### 3.2. Хэш-функция

Для преобразования входных байтов в 32-битное целое используется собственная некриптографическая хэш-функция. Её работа состоит из следующих этапов.

**Инициализация.** Внутреннее состояние устанавливается в константу `0x6A09E667` - дробную часть квадратного корня из 2. Это обеспечивает ненулевое начальное состояние.

**Обработка байтов.** Для каждого байта входных данных последовательно выполняются четыре операции:

1. **XOR с байтом** - внедрение входных данных в текущее состояние.
2. **Прибавление константы** `0x9E3779B9` - «золотая» константа Кнута, равная $\lfloor 2^{32} / \varphi \rfloor$, где $\varphi$ - золотое сечение. Гарантирует равномерное распределение значений при последовательном прибавлении.
3. **Циклический сдвиг влево на 13 бит** - вращение битов слова без потери информации; разрушает линейные зависимости между последовательными состояниями.
4. **XOR со значением, сдвинутым вправо на 16 бит** - смешивает старшие и младшие биты, обеспечивая «лавинный эффект»: изменение одного входного бита затрагивает всё выходное слово.

Все операции выполняются в пространстве 32-битных беззнаковых целых.

### 3.3. Генерация чисел

При инициализации генератора `seed` (или текущее время при его отсутствии) хэшируется и сохраняется как ключ. При каждом запросе нового числа:

1. Счётчик вызовов увеличивается на 1, гарантируя уникальность входа для хэш-функции.
2. Байтовые представления ключа (4 байта) и счётчика (8 байт) конкатенируются в 12-байтовую последовательность.
3. Полученная последовательность хэшируется.
4. Результат делится на максимальное 32-битное значение `0xFFFFFFFF`, что даёт равномерно распределённое вещественное число в $[0, 1]$.

Использование 8-байтового счётчика позволяет сгенерировать до $2^{64} \approx 1{,}8 \times 10^{19}$ уникальных значений без повторения последовательности.

## 4. Встроенный генератор - Вихрь Мерсенна (MT19937)

В качестве эталона используется `random.random()` из стандартной библиотеки Python - реализация алгоритма **Вихрь Мерсенна (Mersenne Twister, MT19937)**, разработанного Мацумото и Нисимурой в 1997 году.

### 4.1. Основные характеристики

| Параметр | Значение |
|----------|----------|
| Период | 2^19937 - 1 (число Мерсенна) |
| Размер внутреннего состояния | 624 слова по 32 бита = 2496 байт |
| Размерность равномерного распределения | 623 |
| Тип | ГПСЧ общего назначения, не криптостойкий |

### 4.2. Внутреннее состояние и инициализация

Генератор хранит массив из **624 32-битных слов** - это и есть «вихрь». При инициализации через `random.seed(s)` массив заполняется по рекуррентной формуле:

```python
state[0] = seed
state[i] = 1812433253 * (state[i-1] XOR (state[i-1] >> 30)) + i 
```

Константа `1812433253` выбрана так, чтобы обеспечить хорошее перемешивание битов на этапе инициализации.

### 4.3. Шаг генерации - «скручивание»

Каждые 624 вызова генератор «перекручивает» весь массив состояния (отсюда название «вихрь»). Для каждой позиции `i'.

Число **397** (сдвиг по массиву) и **19937** (степень числа Мерсенна) - ключевые параметры, подобранные авторами для максимального периода.

### 4.4. Извлечение числа - «закалка» (Tempering)

Сырое значение из массива проходит через четыре побитовые операции, которые улучшают статистические свойства выходной последовательности:

```python
y = state[index]
y ^= (y >> 11)
y ^= (y << 7)  & 0x9D2C5680
y ^= (y << 15) & 0xEFC60000
y ^= (y >> 18)
```

Этот этап называется «закалкой» - он устраняет корреляции, которые были бы заметны в сыром состоянии массива. Итоговое 32-битное целое делится на 2^32, давая вещественное число из [0, 1).

### 4.5. Почему MT19937 используется как эталон

- **Огромный период** (2^19937 - 1) исключает повторение последовательности на практике.
- **623-мерное равномерное распределение** - числа равномерно распределены в пространстве размерностью до 623, что важно для симуляций Монте-Карло.
- **Высокая скорость** - все операции побитовые, без умножений и делений в основном цикле.

Основной недостаток: MT19937 **не является криптостойким** - по 624 последовательным числам можно полностью восстановить внутреннее состояние и предсказать все последующие значения.

## 5. Вычисление статистик

Выборочное среднее и дисперсия вычисляются по формулам:

$$\bar{x} = \frac{1}{n} \sum_{i=1}^{n} x_i$$

$$s^2 = \frac{1}{n} \sum_{i=1}^{n} (x_i - \bar{x})^2$$

Используется смещённая оценка дисперсии (деление на $n$), что при $n = 100\,000$ несущественно отличается от несмещённой.

## 7. Выводы

1. Реализованный генератор показал среднее и дисперсию, близкие к теоретическим значениям для $U(0,1)$. Отклонение не превышает $0{,}001$ по обоим параметрам на выборке из 100 000 элементов - это приемлемый результат для задач статистического моделирования.

2. Встроенный генератор демонстрирует аналогичную точность, что ожидаемо для хорошо изученного алгоритма Вихрь Мерсенна.

3. Архитектура «ключ + счётчик + хэш» обеспечивает детерминированность (при одинаковом `seed` последовательность воспроизводима) и независимость последовательных значений, поскольку хэш-функция маскирует линейный счётчик.

4. Основное ограничение реализованного генератора - используемая хэш-функция не является криптостойкой. Однако для учебных задач её качества достаточно.

5. Таким образом, реализованный датчик случайных чисел удовлетворяет требованиям задания: генерирует числа в $[0, 1]$ с распределением, близким к равномерному, и статистически неотличимым от встроенного генератора на выборке заданного объёма.
