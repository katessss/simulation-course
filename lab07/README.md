# Лабораторная работа №7: Марковская модель погоды

## Теоретические основы

### Марковская цепь

Марковская цепь — это математическая модель случайного процесса, обладающая **марковским свойством**: вероятность перехода в следующее состояние зависит только от текущего состояния и не зависит от предыстории.

**Формальное определение:**

```
P(X_{n+1} = j | X_n = i, X_{n-1} = k, ..., X_0 = m) = P(X_{n+1} = j | X_n = i)
```

Где:
- `X_n` — состояние системы в момент времени `n`
- `i, j, k, m` — возможные состояния

**Интерпретация для модели погоды:**

Если сегодня пасмурно, то вероятность того, что завтра будет ясно, зависит только от того, что сегодня пасмурно, и не зависит от того, какая погода была вчера, позавчера и т.д.

### Пространство состояний

В данной модели используется конечное пространство состояний:

```
S = {1, 2, 3}

где:
  1 — Ясно
  2 — Облачно
  3 — Пасмурно
```

Система в каждый момент времени `t` находится в одном из этих состояний.

### Дискретное время

Переходы между состояниями происходят в дискретные моменты времени `t = 0, 1, 2, 3, ...`

В нашей модели единица времени равна одному дню.

## Математическая модель

### Матрица переходов

Матрица переходов `P` — это стохастическая матрица размером `n × n` (где `n` — количество состояний), элемент `P[i,j]` которой определяет вероятность перехода из состояния `i` в состояние `j`.

**Пример матрицы переходов:**

```
                  Следующее состояние
                  ┌─────────────────────┐
                  │ Ясно  Облачно  Пасмурно
        ┌─────────┼─────────────────────┤
Текущее │ Ясно    │ 0.70    0.20    0.10
состоян.│ Облачно │ 0.30    0.40    0.30
        │ Пасмурно│ 0.20    0.30    0.50
        └─────────┴─────────────────────┘

P = [ 0.7  0.2  0.1 ]
    [ 0.3  0.4  0.3 ]
    [ 0.2  0.3  0.5 ]
```

**Интерпретация элементов:**

- `P[0,0] = 0.7` — Если сегодня ясно, то завтра будет ясно с вероятностью 70%
- `P[0,1] = 0.2` — Если сегодня ясно, то завтра будет облачно с вероятностью 20%
- `P[0,2] = 0.1` — Если сегодня ясно, то завтра будет пасмурно с вероятностью 10%

И так далее для остальных строк.

### Свойства стохастической матрицы

Матрица переходов должна удовлетворять двум условиям:

**1. Неотрицательность:**

```
P[i,j] ≥ 0  для всех i, j
```

Все вероятности должны быть неотрицательными.

**2. Нормировка (стохастичность):**

```
Σ_j P[i,j] = 1  для всех i
```

Сумма элементов в каждой строке равна единице.

**Проверка в коде:**

```python
def _validate_matrix(self):
    """Проверка корректности матрицы переходов"""
    
    # Проверка размера
    if self.transition_matrix.shape != (3, 3):
        raise ValueError("Матрица переходов должна быть 3x3")
    
    # Проверка стохастичности (сумма строк = 1)
    row_sums = self.transition_matrix.sum(axis=1)
    if not np.allclose(row_sums, 1.0):
        raise ValueError("Сумма вероятностей в каждой строке должна быть равна 1")
    
    # Проверка неотрицательности
    if np.any(self.transition_matrix < 0):
        raise ValueError("Все вероятности должны быть неотрицательными")
```

### Стационарное распределение

Стационарное (или предельное) распределение `π` — это вероятностное распределение, которое не изменяется при применении матрицы переходов.

**Математическое определение:**

```
π P = π
```

где `π` — вектор-строка вероятностей, удовлетворяющий условию:

```
π_1 + π_2 + ... + π_n = 1
```

**Смысл:**

После достаточно большого числа шагов система "забывает" своё начальное состояние, и распределение вероятностей сходится к стационарному. Это означает, что в долгосрочной перспективе доля времени, проведённого в каждом состоянии, стремится к соответствующему элементу стационарного распределения.

**Вычисление стационарного распределения:**

Стационарное распределение является собственным вектором транспонированной матрицы `P^T`, соответствующим собственному значению `λ = 1`.

**Реализация в коде:**

```python
def get_stationary_distribution(self):
    """
    Вычисление теоретического стационарного распределения
    
    Returns:
        numpy array с вероятностями для состояний [1, 2, 3]
    """
    try:
        # Решаем систему πP = π
        eigenvalues, eigenvectors = np.linalg.eig(self.transition_matrix.T)
        
        # Находим собственный вектор для собственного значения 1
        idx = np.argmin(np.abs(eigenvalues - 1))
        stationary = np.real(eigenvectors[:, idx])
        stationary = stationary / stationary.sum()
        
        return stationary
    except Exception as e:
        print(f"Ошибка вычисления стационарного распределения: {e}")
        return np.array([1/3, 1/3, 1/3])
```

**Пример:**

Для матрицы из примера выше стационарное распределение:

```
π ≈ [0.4565, 0.2826, 0.2609]
```

Это означает, что при длительной симуляции:
- Примерно 45.65% дней будут ясными
- Примерно 28.26% дней будут облачными
- Примерно 26.09% дней будут пасмурными

## Реализация

### Алгоритм симуляции одного шага

На каждом шаге симуляции выполняются следующие действия:

**Шаг 1:** Определить текущее состояние `i`

**Шаг 2:** Извлечь соответствующую строку из матрицы переходов:

```
probabilities = P[i, :]
```

**Шаг 3:** Выполнить случайный выбор следующего состояния согласно распределению вероятностей:

```
j = random_choice({1, 2, 3}, probabilities)
```

**Шаг 4:** Обновить текущее состояние:

```
current_state = j
```

**Шаг 5:** Сохранить состояние в историю

**Реализация в коде:**

```python
def step(self):
    """
    Выполнить один шаг симуляции
    
    Returns:
        новое состояние (1, 2, 3)
    """
    # Выбираем следующее состояние на основе текущего
    probabilities = self.transition_matrix[self.current_state - 1]
    next_state = np.random.choice([1, 2, 3], p=probabilities)
    
    self.current_state = next_state
    self.history.append(next_state)
    self.step_count += 1
    
    return next_state
```

### Эмпирическое распределение

Эмпирическое распределение вычисляется на основе наблюдаемой истории состояний:

```
empirical[i] = count(state == i) / total_steps
```

**Реализация в коде:**

```python
def get_empirical_distribution(self):
    """
    Получить эмпирическое распределение из истории
    
    Returns:
        numpy array с частотами состояний [1, 2, 3]
    """
    if not self.history:
        return np.array([0.0, 0.0, 0.0])
    
    counter = Counter(self.history)
    total = len(self.history)
    
    return np.array([
        counter.get(1, 0) / total,
        counter.get(2, 0) / total,
        counter.get(3, 0) / total
    ])
```

### Сходимость к стационарному распределению

При увеличении числа шагов эмпирическое распределение сходится к стационарному. Это свойство называется **эргодичностью** марковской цепи.

**Условия эргодичности:**

1. **Неразложимость:** Из любого состояния можно попасть в любое другое за конечное число шагов
2. **Апериодичность:** Не существует циклов фиксированной длины

Наша модель погоды удовлетворяет обоим условиям при условии, что все элементы матрицы `P` строго положительны или система имеет апериодическую структуру.
